# Name Resolution

## 1. Resolution Priority
当前 market 模块的标的解析优先级如下：

1. **标准代码优先**
   - 例如：`HK-00700`
   - 例如：`US-AAPL`
2. **裸代码归一化**
   - 例如：`00700` → `HK-00700`
   - 例如：`AAPL` → `US-AAPL`
3. **本地名称映射**
   - 例如：`腾讯` → `HK-00700`
   - 例如：`苹果` → `US-AAPL`
4. **整句清洗后再尝试识别**
   - 去掉“看 / 查 / 最近 / K线 / 资金流向 / 走势”等修饰词后再尝试归一化
5. **歧义澄清**
   - 若名称同时存在港股和美股且用户未明确市场，则先返回澄清提示

## 2. Standard Symbol Format
统一标准代码格式：
- 港股：`HK-00700`
- 美股：`US-AAPL`

当前 skill 接收：
- 标准代码
- 裸代码
- 常见中文名称
- 常见英文名称

执行前统一归一化到标准代码格式。

## 4. HK / US Ambiguous Names
当前已支持的双市场歧义名称包括：

| Name | HK | US |
|---|---|---|
| 阿里 / 阿里巴巴 | `HK-09988` | `US-BABA` |
| 百度 | `HK-09888` | `US-BIDU` |
| 京东 | `HK-09618` | `US-JD` |
| 哔哩哔哩 | `HK-09626` | `US-BILI` |
| 蔚来 | `HK-09866` | `US-NIO` |
| 小鹏 | `HK-09868` | `US-XPEV` |
| 理想 | `HK-02015` | `US-LI` |

处理规则：
- 用户明确说“港股” / “美股”时，直接落到对应市场
- 用户输入标准代码时，直接落到对应标的
- 用户未明确市场时，先进入澄清态

## 5. Sentence Cleaning Strategy
当用户输入整句时，路由层会尝试去掉一些常见修饰词，例如：
- 看 / 看一下 / 看看
- 查 / 查一下 / 查询
- 最近
- K线 / 日K / 周K / 月K / 分钟K
- 资金流向 / 主力资金
- 盘口 / 摆盘
- 市场状态
- 基础信息
- 走势

然后再对剩余文本尝试做 symbol 归一化。

## 6. Current Limits
当前名称解析仍有这些限制：
- 仍以本地映射表为主，不是真正的证券搜索接口
- 没进入映射表的中文名称，不保证一定能识别
- 路由层能识别名称，不代表后端当前环境一定支持该标的 / 周期组合

## 7. Example
### 可直接识别
- `看腾讯最近10根日K`
- `看苹果行情`
- `查 US-AAPL 最新价`

### 需要歧义澄清
- `看阿里巴巴行情`
- `看百度K线`

## 8. Future Direction
后续更理想的方向是：
- 接入正式的证券搜索 / 标的匹配接口
- 减少纯本地映射带来的覆盖范围限制
- 支持更多中文名称、别名和模糊匹配
