# Edge Cases

## 1. 港股 / 美股名称歧义
部分中文股票名称同时对应港股和美股，例如：
- 阿里巴巴：`HK-09988` / `US-BABA`
- 百度：`HK-09888` / `US-BIDU`
- 京东：`HK-09618` / `US-JD`
- 哔哩哔哩：`HK-09626` / `US-BILI`
- 蔚来：`HK-09866` / `US-NIO`
- 小鹏：`HK-09868` / `US-XPEV`
- 理想：`HK-02015` / `US-LI`

处理规则：
- 若用户已明确说“港股”或“美股”，则直接路由到对应市场。
- 若用户未明确市场，则先返回澄清提示，不默认猜测。

## 2. 代码格式规范
- 港股统一格式：`HK-00700`
- 美股统一格式：`US-AAPL`
- 输入允许中文名称、裸代码或标准代码，执行时统一转标准格式。

## 3. 资金流向尾部 0 值记录
`capital_flow` 接口在部分标的上，原始返回末尾可能带有全 0 占位记录。

已验证现象：
- `HK-09988` 在 `num=5` 时，原始返回可能是 6 条，其中最后 1 条为全 0。
- `HK-09988` 在 `num=10` 时，原始返回可能是 8 条。
- `HK-00700` 在 `num=10` 时，原始返回可正常返回 10 条，且最新一条为有效数据。

处理规则：
- 默认忽略尾部全 0 占位记录。
- 对外展示有效条数，并保留原始条数字段供开发调试使用。

## 4. detail mode 默认展示上限
当前 detail mode 适用于：
- 资金流向
- K线
- 分时

处理规则：
- 默认最多展示最近 10 条记录。
- 若实际记录超过 10 条，文本中明确提示“仅展示最近 10 条”。

## 5. K线返回条数依赖后端实际数据
即使传入 `num=5` 或 `num=20`，后端实际返回条数仍可能少于请求值。

已验证现象：
- `HK-00700` 某些 K线查询场景下，即使请求多条，当前环境实测只返回 1 条。

处理原则：
- skill 负责按实际返回做摘要或明细展示。
- 不假设后端一定严格返回请求条数。

## 6. 美股 K线当前环境可能返回 404
已验证现象：
- `看苹果5分钟K`
- `看英伟达最近5根周K`

当前路由可正确识别意图与标的，但后端在当前环境下返回 `404`。

处理原则：
- 对用户提示“当前环境下，该标的或该查询参数暂不可用（接口返回 404）”。
- 这属于当前环境限制，不代表路由逻辑本身错误。

## 7. 板块 / 期权能力在当前环境未正式开放
已验证现象：
- `/quote/get_plate_list`：生产环境暂不支持
- `/derivatives/get_option_chain`：生产环境暂不支持
- `/derivatives/get_option_expiration_date`：404

处理原则：
- 这些能力仅保留为未来扩展接口，不作为当前一期主承诺能力。

## 8. 安装成功不等于可真实查询
当前 skill 安装后，还必须配置：
- `NOAH_API_BASE_URL`
- `NOAH_MARKET_APIKEY`

否则：
- skill 可以被识别 / 加载
- 但不能执行真实行情查询

## 9. 不要把 GitHub token 当成行情 API key
已出现过用户把 `ghp_...` GitHub token 错当成 `NOAH_MARKET_APIKEY` 的情况。

处理原则：
- 必须使用公司证券行情服务 API key。
- 文档和错误提示中应明确禁止把 GitHub token / OpenClaw token / 其他平台凭证用于行情接口鉴权。
